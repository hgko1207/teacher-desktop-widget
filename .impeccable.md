# Teacher's Desk — Design Context

## Design Context

### Users
**대상**: 초/중/고등학교 교사 (아내 포함, 한국)
**사용 맥락**: 수업 중 및 하루 종일 켜놓는 항상-온(always-on) 데스크탑 위젯.
원래 목적은 바탕화면에 박제(wallpaper-level)하는 것이었으나, 현재는 일반 창으로 동작.
모니터 구석이나 보조 모니터에 상시 노출 — 사용자가 능동적으로 조작하기보다 **흘낏 훑어보는** 패턴.

**Job to be done**:
- 지금 몇 교시인지, 다음 수업까지 얼마나 남았는지 즉시 확인
- 오늘 급식, 학사일정, D-Day 체크
- 할 일/메모 간단히 확인·추가
- 수업 중 타이머·랜덤뽑기 등 도구 바로 접근

### Brand Personality
**3단어**: 차분한 · 전문적인 · 군더더기 없는 (Calm · Professional · Uncluttered)

**감성 목표**: 교사가 하루 8시간 이상 켜둬도 피로하지 않은 배경.
화려함이나 유머보다 **신뢰감과 정보 명료성**이 우선.
노트북/PC 바탕화면과 자연스럽게 어우러지는 도구적 존재감.

### Aesthetic Direction
**시각 톤**: Glassmorphism light — 반투명 카드(rgba 0.8) + backdrop-blur.
화사하지 않은 파스텔 배경, 정보는 진한 글씨로 또렷하게.

**테마 시스템**: 라이트 모드 전용 3종 (인디고/벚꽃/민트).
사용자가 취향에 맞게 선택하되, 어떤 테마도 눈에 거슬리지 않아야 함.

**레퍼런스**: 없음 (사용자가 명확한 레퍼런스 없음 — 기존 코드베이스가 레퍼런스).
**안티 레퍼런스**:
- 게임 UI (과도한 애니메이션, 네온, 어두운 다크 모드)
- 앱 광고/쇼케이스 스타일 (화려한 그라디언트, 대형 히어로 이미지)
- 어린이용 앱 느낌 (둥글둥글한 귀여운 아이콘, 강한 채도)

### Accessibility Analysis
코드베이스 분석 기반 접근성 현황 및 권장사항:

**현재 상태**:
- 기본 폰트 14px — 데스크탑 근거리 시청에 적합
- Pretendard 폰트 — 한국어 가독성 우수
- 상태 색상(미세먼지 등)에 텍스트 레이블 병기 → 색약 친화적
- 글래스모피즘 0.8 불투명도 — 배경 이미지에 따라 가독성 저하 가능

**개선 권장**:
- 시간표 과목 배경색(파스텔) + 어두운 텍스트 조합 유지 필수 (밝은 배경에 흰 글씨 금지)
- 위젯 제목(11px) 등 초소형 텍스트는 font-weight 600+ 유지
- 클릭 영역(버튼/아이콘) 최소 32×32px 확보
- 감속 모션 설정(`prefers-reduced-motion`) 고려 — 현재 애니메이션 최소화로 사실상 양호

### Design Principles

1. **항상-온을 위한 절제 (Ambient Restraint)**
   위젯은 주인공이 아닌 배경이다. 시선을 끌어당기는 대신, 필요할 때 정보를 내어준다.
   애니메이션·강조·알림은 최소한으로.

2. **정보 계층 우선 (Information Hierarchy First)**
   지금 교시 > 시계 > 날씨·미세먼지 > 나머지 순서.
   중요한 정보가 중요하게 보여야 한다. 장식은 정보 계층을 방해하지 않는다.

3. **전문적 일관성 (Professional Consistency)**
   교사는 매일 같은 화면을 본다. 컴포넌트 스타일·간격·색상이 어디서나 예측 가능해야 한다.
   카드 스타일, 패딩, 폰트 크기 — 전체를 통일한다.

4. **한국 교육 맥락 최우선 (Korean Education Context)**
   교시/급식/학사일정/컴시간 — 모든 텍스트와 데이터 표현이 국내 교사에게 자연스러워야 한다.
   영어 레이블보다 한국어 레이블 우선.

5. **데스크탑 네이티브 (Desktop Native Feel)**
   웹/모바일 앱과 달리, 창 프레임 없이 바탕화면과 어우러지는 느낌이 목표.
   경계가 흐릿한 글래스모피즘 스타일 유지, 과도한 그림자 지양.

## Technical Notes (Design-Relevant)

- **바탕화면 고정 모드**: `windowPin.ts`에 koffi + Windows API로 구현됨. 현재 동작 중.
  고정 모드 전환 시 UI 피드백이 명확해야 함 (현재 상태 인디케이터 필요 여부 검토).
- **항상-온 최적화**: 메모리·GPU 사용 최소화 — 불필요한 리렌더, 큰 이미지, 복잡한 CSS 지양.
- **창 크기**: 고정 크기 레이아웃 (130px 상단 + 하단 flex). 반응형 불필요.
