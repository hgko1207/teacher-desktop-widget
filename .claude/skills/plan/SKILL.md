사용자의 요청사항을 프로젝트 플랜에 반영합니다.

1. 사용자의 요청 내용을 분석합니다: $ARGUMENTS
2. 현재 플랜 파일(plan.md)을 읽습니다.
3. 요청 유형에 따라 처리합니다:
   - **새 기능/작업 요청**: "다음 작업 (구현 예정)" 섹션에 추가
     - 각 항목에 제목, 구현 내용, 기술 요구사항, 우선순위를 명확히 작성
   - **완료 보고**: 해당 항목을 "완료된 작업" 섹션으로 이동 (완료일 기록)
   - **변경 요청**: 기존 항목 수정 또는 우선순위 재조정
   - **상태 확인**: 현재 플랜 요약 제공
4. 중복 항목은 병합하고, 관련 항목은 같은 Phase에 그룹핑합니다.
5. research.md의 내용과 일관성을 유지합니다.
6. 사용자에게 정리된 플랜을 간단히 요약합니다.
