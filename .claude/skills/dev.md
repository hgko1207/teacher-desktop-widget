## 개발 규칙 (코딩 표준)

### TypeScript 엄격 모드
- `any`, `unknown` 타입 사용 금지 - 반드시 구체적 타입 정의
- `strict: true` 설정 필수
- 모든 함수의 매개변수와 반환 타입 명시
- 타입 단언(`as`) 최소화, 타입 가드 우선 사용

### 타입 체크
- 작업 단계 완료 시마다 `npm run typecheck` 실행
- 새로운 타입 에러를 만들지 않음
- 기존 에러 수를 늘리지 않음

### 코드 품질
- 사용하지 않는 import/변수 금지
- console.log 디버깅 코드 커밋 금지
- 매직 넘버 대신 상수 사용
- 컴포넌트는 단일 책임 원칙

### 프로젝트 진행
- 모든 작업 완료 시 plan.md에서 해당 항목을 완료로 표시
- 중간에 멈추지 않고 Phase 단위로 완료
- 에러 발생 시 즉시 수정 후 진행

### Git
- 의미 있는 단위로 커밋 (Phase 또는 위젯 단위)
- 커밋 메시지는 한글 또는 영어로 명확하게
